# Однослойный Перцептрон (Single-layer Perceptron)

Работа представляет собой реализацию однослойного перцептрона с нуля на языке Python с использованием библиотеки `NumPy` (без использования готовых решений из высокоуровневых ML фреймворков для самой модели). Проект предназначен для изучения базовых принципов работы нейронных сетей, методов оптимизации, регуляризации и оценки моделей бинарной классификации.

## Структура проекта и принцип работы компонентов

Проект разделен на несколько логических модулей, каждый из которых выполняет свою специфическую задачу.

### 1. Ядро модели: `perceptron.py`
Содержит класс `Perceptron`, реализующий логику работы однослойной нейронной сети.
- **Инициализация весов (`init_type`)**: Поддерживается инициализация нулями (`zeros`), малыми случайными числами (`small`) и большими случайными числами (`large`). Это позволяет исследовать влияние начальных весов на стабильность и скорость сходимости модели.
- **Прямой проход (`forward`)**: реализован через вычисление линейной комбинации признаков и весов с последующим применением сигмоиды. Добавлена математическая защита от переполнения экспоненты с помощью `np.clip`.
- **Функции потерь (`loss_type`)**: 
  - `bce` (Binary Cross-Entropy) — классическая логистическая функция потерь.
  - `hinge` (Hinge Loss) — функция потерь, используемая в методе опорных векторов (SVM), акцентирующая внимание на максимизации отступа (margin).
- **Оптимизация (Momentum SGD)**: обучение (метод `fit`) происходит с помощью стохастического градиентного спуска с поддержкой мини-батчей (Mini-batch SGD) и импульса (Momentum). Momentum помогает сглаживать осцилляции при спуске и позволяет быстрее и увереннее сходиться к минимуму функции потерь.
- **L2-регуляризация (`l2_lambda`)**: реализован штраф за большие веса для предотвращения переобучения (overfitting). Градиенты пересчитываются с учетом производной от регуляризационного члена, штрафуя веса пропорционально их величине.

### 2. Вспомогательные инструменты: `utils.py`
Содержит утилиты для визуализации, генерации данных и оценки модели:
- `plot_decision_boundary`: строит графики разделяющей поверхности (decision boundary), что позволяет наглядно оценить, как модель проводит границу между классами в двумерном пространстве.
- `generate_gaussian_clouds` и `generate_xor`: функции для генерации синтетических датасетов. Гауссовы облака демонстрируют работу модели на линейно разделимых данных, в то время как датасет XOR позволяет наглядно показать фундаментальное ограничение однослойного перцептрона — невозможность разделить нелинейно разделимые данные.
- `custom_cv`: реализация K-Fold кросс-валидации (по умолчанию 5-Fold) "с нуля" для подбора оптимальных гиперпараметров (скорости обучения и размера батча) без переобучения на тестовой выборке.

### 3. Экспериментальная среда (Точка входа): `main.py`
Главный скрипт, оркестрирующий весь процесс обучения и тестирования модели. В нем последовательно выполняются:
- **Подготовка данных**: генерация основного датасета через `sklearn.datasets.make_classification`, разбиение на обучающую и тестовую выборки (`train_test_split`) и стандартизация признаков (`StandardScaler`).
- **Базовое обучение**: обучение модели со стандартными параметрами и первичная оценка точности (Accuracy). Сохранение графиков кривой обучения и разделяющей границы.
- **Эксперименты с гиперпараметрами**: исследование влияния `learning rate` (скорости обучения), `batch size` (размера батча) и инициализации весов на финальный результат. Результаты логируются в CSV-файл.
- **Дополнительные эксперименты**:
  - Тестирование модели на датасетах Гауссовых облаков и XOR.
  - Сравнение сходимости при использовании различных функций потерь (BCE и Hinge).
  - Изучение влияния коэффициента L2-регуляризации на норму весов (демонстрация механизма сдерживания роста весов) и качество обобщения (test accuracy).
  - Расчет комплексных метрик классификации (Precision, Recall, F1-score, ROC-AUC) и построение ROC-кривой. Отдельно визуализируются ошибки модели на тестовой выборке.
  - Демонстрация влияния коэффициента `momentum` на скорость и стабильность сходимости.
  - Запуск кастомной кросс-валидации для автоматического поиска лучших параметров.

## Установка и запуск

### Требования
- Установленный **Python 3.8** (или выше).

### Шаги по запуску

1. **Склонируйте репозиторий или скачайте файлы проекта** в отдельную папку.
2. **Откройте терминал** (или командную строку) и перейдите в директорию проекта.

3. **Создайте виртуальное окружение (рекомендуется):**
   ```bash
   python -m venv venv
   ```

4. **Активируйте виртуальное окружение:**
   - На Windows (cmd/PowerShell):
     ```bash
     venv\Scripts\activate
     ```
   - На macOS / Linux:
     ```bash
     source venv/bin/activate
     ```

5. **Установите зависимости:**
   Все необходимые внешние библиотеки (numpy, pandas, matplotlib, sklearn) описаны в файле `requirements.txt`.
   ```bash
   pip install -r requirements.txt
   ```

6. **Запустите основной скрипт:**
   ```bash
   python main.py
   ```

### Результаты работы скрипта
В процессе выполнения `main.py` будет выводить в консоль логи обучения, расчетные метрики, промежуточные результаты кросс-валидации и статус выполнения различных экспериментов.

По завершении (и в процессе) в корне проекта будет создана директория **`results/`**, в которой будут сохранены все визуализации и таблицы:
- `basic_training.png` — графики кривой обучения (Loss) и разделяющей границы для базовой модели.
- `experiments_results.csv` — таблица с результатами влияния гиперпараметров на финальную точность.
- `init_experiment.png` — график влияния способа инициализации весов на сходимость.
- `gauss_experiment.png` и `xor_experiment.png` — разделяющие границы на дополнительных синтетических датасетах (проверка линейной/нелинейной разделимости).
- `loss_comparison.png` — сравнение скорости падения ошибки для BCE и Hinge loss.
- `l2_regularization_effect.png` — влияние L2-регуляризации на штраф весов (их норму) и качество модели.
- `metrics_and_errors.png` — визуализация конкретных точек, на которых модель ошиблась, а также ROC-кривая (и значение ROC-AUC).
- `momentum_experiment.png` — график сходимости потерь при различных значениях параметра Momentum (демонстрация сглаживания и ускорения обучения).
